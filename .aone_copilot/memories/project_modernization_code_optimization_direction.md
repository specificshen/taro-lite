工程已经收敛成 8 个包，下一阶段重点是代码优化：建设全现代化工程，让底座源码更容易理解；文件导入导出尽可能使用 ESM；重新梳理目录规范并合理组织包目录；目录名与文件名优先使用 kebab-case。

Why: 用户希望后续工作不只是包收敛，还要提升源码可读性、现代化程度和长期维护体验。
How to apply: 涉及代码重构、导入导出、目录调整、文件命名和包内结构设计时，优先按全现代化、ESM-first、kebab-case 和清晰包目录组织来建议与实施。
